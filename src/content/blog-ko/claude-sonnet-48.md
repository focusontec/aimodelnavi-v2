2026년 3월 31일, Claude Code의 npm 패키지(v2.1.88)에서 **"sonnet-4-8"**이라는 문자열이 발견되었다.

Anthropic은 공식 발표하지 않았지만, 유출된 정보를 통해 Sonnet 4.8의 윤곽은 상당히 뚜렷해지고 있다.

---

## 버전 번호의 수수께끼: 왜 4.7을 건너뛰고 4.8인가

Anthropic의 Sonnet 계열은 그동안 일정한 간격으로 버전을 업데이트해 왔다:

| 모델 | 출시일 | API 문자열 |
|--------|---------|----------|
| Claude 3.5 Sonnet | 2024년 6월 | claude-3-5-sonnet-20240620 |
| Claude 3.7 Sonnet | 2025년 2월 | claude-3-7-sonnet-20250219 |
| Claude 4 Sonnet | 2025년 5월 | claude-sonnet-4-0 |
| Claude Sonnet 4.5 | 2025년 9월 | claude-sonnet-4-5 |
| Claude Sonnet 4.6 | 2026년 2월 | claude-sonnet-4-6 |
| **Claude Sonnet 4.8** | **2026년 5월(예상)** | claude-sonnet-4-8 |

4.7을 건너뛰고 4.8로 간다는 점은 **Opus 4.7의 단순한 이식이 아님**을 시사한다. 서로 다른 타임라인, 서로 다른 학습 런으로 개발되었을 가능성이 크다.

---

## 예상 스펙

유출 정보와 Opus 4.7의 성능 개선폭을 기반으로 한 전망:

| 항목 | Sonnet 4.6(현재) | Sonnet 4.8(예상) |
|------|-------------------|-------------------|
| SWE-bench Verified | 79.6% | **82~84%** |
| GPQA Diamond | 74.1% | **76~78%** |
| 이미지 해상도 상한 | ~1.25MP | **3.75MP**(3배) |
| 컨텍스트 윈도우 | 1M(베타) | 1M(GA 가능성) |
| 가격(입력/출력) | $3/$15 | $3/$15(변경 없을 가능성) |
| Knowledge cutoff | 2025년 8월 | 2025년 말~2026년 초 |

### SWE-bench Verified: 82~84%의 근거

Sonnet 4.6는 SWE-bench Verified에서 79.6%를 기록했으며, Opus 4.6(80.8%)과 거의 동등한 수준이었다. Opus 4.7은 87.6%에 도달하며 Opus 4.6 대비 약 7포인트의 개선을 보였다.

Sonnet 4.8이 동일한 수준의 개선폭(6~8포인트)을 Sonnet 4.6에서 달성한다면, **82~84%**는 충분히 현실적인 수치다.

참고로 2026년 5월 25일 기준 SWE-bench Verified 랭킹 상위권:

| 순위 | 모델 | 점수 |
|------|--------|--------|
| 1 | Claude Mythos Preview | 93.9% |
| 2 | Claude Opus 4.7 | 87.6% |
| 3 | GPT-5.3 Codex | 85.0% |
| 4 | Claude Opus 4.5 | 80.9% |
| 5 | Claude Opus 4.6 | 80.8% |
| 5 | DeepSeek V4 Pro (Max) | 80.6% |
| 7 | Gemini 3.1 Pro | 80.6% |
| 8 | Kimi K2.6 | 80.2% |
| 8 | MiniMax M2.5 | 80.2% |
| 10 | Claude Sonnet 4.6 | 79.6% |

Sonnet 4.8이 82~84%를 달성하면, **Opus 4.5·Opus 4.6을 넘어 Opus 4.7에 근접**하게 된다. Sonnet 티어($3/$15) 가격으로 Opus급 코딩 성능을 가진 모델이 등장하는 셈이다.

### Vision: 3.75MP 이미지 대응

Opus 4.7은 98.5%의 시각 정확도와 3.75MP 이미지 해상도를 달성했다. Sonnet 4.8은 이 능력을 미드 티어 가격대로 가져올 것으로 예상된다.

Sonnet 4.6의 약 1.25MP에서 3.75MP로의 **3배 해상도 향상**은 문서 분석, UI 목업 이해, 차트 판독 등의 실용성을 대폭 향상시킨다.

### 새로운 'xhigh' 에포트 레벨

Opus 4.7에서 도입된 **xhigh** 에포트 레벨(high와 max 사이)이 Sonnet 4.8에도 이식될 것으로 보인다.

현재 에포트 레벨: low / medium / high / max

여기에 xhigh가 추가됨으로써 비용과 성능 간의 트레이드오프를 더 세밀하게 제어할 수 있게 된다.

### 토크나이저 업데이트

Opus 4.7의 새 토크나이저는 콘텐츠 유형에 따라 1.0~1.35배의 토큰을 생성한다:

| 콘텐츠 유형 | 토큰 증가율 |
|----------------|-------------|
| 영어 산문 | ~1.0배(변화 없음) |
| 코드 | ~1.1~1.2배 |
| 구조화 데이터(JSON, XML) | 최대 1.35배 |

Sonnet 4.8이 이 토크나이저를 채택하면, **실질적인 비용이 10~35% 상승할 가능성이 있다**. API 가격은 $3/$15를 유지하더라도 동일한 콘텐츠가 더 많은 토큰으로 분할되기 때문이다.

## Sonnet 4.6 대비 주요 개선점

### 1. Vision 능력 대폭 강화

Opus 4.7에서 계승될 것으로 보이는 시각 능력:
- 이미지 해상도: 1.25MP → 3.75MP
- 시각 정확도: 98.5%(Opus 4.7 실측)
- 문서 스크린샷, UI 목업, 차트의 고정밀 이해

### 2. 코딩 성능 향상

Opus 4.7의 개선 실적:
- CursorBench: 58% → 70%(+12포인트)
- 93개 태스크 코딩 벤치마크: +13%
- 프로덕션 환경 태스크 해결 수: 3배

Sonnet 4.8은 이러한 개선을 $3/$15 가격대에 가져온다.

### 3. 프롬프트 해석 방식의 변화

Opus 4.7은 "4.6보다 더 문자 그대로 프롬프트를 해석한다". Sonnet 4.8도 유사한 행동 변화가 예상된다. 더 정확한 지시 실행이 가능하지만, 모호한 지시에 대한 유연성이 낮아질 수도 있다.

## 가격: Sonnet 티어의 안정성

Sonnet 계열 가격은 3.5부터 4.6까지 일관되게 $3/$15를 유지해 왔다:

| 모델 | 입력/1M | 출력/1M |
|--------|---------|--------|
| Claude 3.5 Sonnet | $3 | $15 |
| Claude 4 Sonnet | $3 | $15 |
| Claude Sonnet 4.5 | $3 | $15 |
| Claude Sonnet 4.6 | $3 | $15 |
| Claude Sonnet 4.8(예상) | $3 | $15 |

한편 Opus 티어는 4.6에서 $5/$25로 변경되었다(기존 $15/$75에서 대폭 인하). Sonnet 4.8이 $3/$15를 유지한다면, **Opus 4.7 가격의 60%로 82~84%의 SWE-bench 성능**을 가진 모델이 탄생하는 셈이다.

## 개발자들의 기대

Sonnet 4.8에 대한 개발자 커뮤니티의 기대를 정리하면:

1. **더 긴 신뢰성 높은 출력 생성**
2. **과도한 엔지니어링 감소** (불필요한 복잡성 추가 억제)
3. **Time-to-First-Token(TTFT) 개선**
4. **대규모 코드베이스에서의 멀티파일 인식**
5. **안정적인 JSON/구조화 출력 및 제약 조건 디코딩(constrained decoding)**

## 결론

Claude Sonnet 4.8은 Anthropic Sonnet 계열 역사상 가장 큰 도약이 될 가능성이 있다.

Opus 4.7의 Vision 능력, 코딩 성능, 에포트 레벨을 $3/$15 가격대로 가져온다. SWE-bench Verified에서 82~84%를 달성하면, **Opus급 성능을 Sonnet 가격으로 제공하는 모델**이 탄생하는 셈이다.

유출 정보에 기반한 전망이지만, Anthropic Sonnet 계열의 역사를 돌이켜보면 이 정도의 개선은 충분히 현실적이다. 공식 발표를 기다릴 수밖에 없지만, 개발자라면 가장 주목해야 할 모델 중 하나임은 분명하다.
